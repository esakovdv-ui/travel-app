# Мои Путешествия — Обзор проекта

Туристический маркетплейс на базе white-label платформы Level Travel.
Пользователь ищет тур → бронирует через WL → заказ сохраняется в CRM и БД → отображается в личном кабинете.

## Ключевые компоненты

### Layout
- **SiteHeader** — фиксированный хедер с 3 состояниями: expanded (поиск развёрнут), compact pill (при скролле), search-open (клик по пилюле). На `/tours` и `/hotels` поиск показывается всегда.
- **SiteFooter** — подвал с контактами и навигацией
- **ConditionalFooter** — скрывает футер на `/tours` и `/hotels` (fullscreen-режим с картой). Использует `usePathname`.
- **BrandLogo** — SVG-иконка + текст "МОИ ПУТЕШЕСТВИЯ" (Unbounded), на мобиле текст скрыт

### Поиск
- **SearchBar** — единый компонент поисковой строки, используется и в хедере и на страницах результатов. Два таба: «Отели» (без поля откуда) и «Туры» (с полем откуда).
- **DestinationSearch** — автокомплит по направлениям из статической базы
- **DateRangePicker** — двойной календарь с выбором диапазона дат и гибкостью (±1 / ±2 дня)

### Tours / Hotels — страницы результатов

Layout страниц `/tours` и `/hotels` выстроен по образцу Яндекс Путешествий:
- Полная ширина страницы, три колонки: **сайдбар фильтров** | **список карточек** | **фиксированная карта**
- Карта занимает `42vw` правой стороны (`position: fixed`), страница компенсирует `padding-right: calc(42vw + 48px)`
- Сайдбар `position: sticky`, список прокручивается независимо
- На планшетах карта скрывается (`≤1024px`), на мобиле — сайдбар тоже (`≤860px`)

**Компоненты:**

- **HotelCard** — карточка отеля. Два варианта:
  - `card` — вертикальный (для главной страницы, каруселей)
  - `row` — горизонтальный (для страниц результатов, Яндекс-стиль)
  - Поддерживает `onMouseEnter` / `onMouseLeave` для связки с картой
  - Hover: синий outline `2px solid var(--c-blue)`, без transform и увеличения тени

- **FiltersPanel** — сайдбар фильтров. Поддерживаемые фильтры:
  - Диапазон цены (priceMin / priceMax)
  - Звёзды (мультиселект)
  - Минимальный рейтинг
  - Питание (BB, HB, AI, UAI...)
  - Регион
  - Линия пляжа
  - Тип пляжа
  - Мгновенное подтверждение
  - Показывает счётчики `filteredCount из totalCount`

- **HotelMap** — интерактивная карта Leaflet:
  - Тайлы CartoDB Voyager (чистые, похожи на Google Maps)
  - Маркеры — ценники (`45 000 ₽`), белые в покое → синие при активации
  - При наведении на карточку — соответствующий маркер подсвечивается
  - `FitBounds` — при загрузке карта автоматически подстраивается под все отели
  - При клике на маркер — попап с мини-карточкой: фото, название, регион, звёзды, цена, кнопка «Смотреть»
  - Динамический импорт с `ssr: false` (Leaflet требует браузер)
  - Координаты: поля `lat` и `long` в объекте отеля от Level Travel API

### Карточки и контент
- **PackageCard** — карточка тура/пакета
- **CategoryCard** — тематическая карточка (категория направлений)
- **ReviewList** — список отзывов

### UI-утилиты
- **Preloader** — компонент загрузки (для состояний загрузки, не для старта приложения)
- **EmptyState** — заглушка при отсутствии результатов

## Level Travel API (`src/lib/leveltravel.ts`)

Версия: `v3.7`. Флоу поиска:

```
enqueueSearch() → pollUntilComplete() → getHotels()
```

Поддерживаются два формата дат:
- **Вариант А**: `start_date` + `nights` (конкретная дата)
- **Вариант Б**: `start_date_from/till` + `end_date_from/till` (диапазон, текущий)

`search_type`: `package` (тур с перелётом) | `hotel` (только отель).

Координаты отеля: поля `lat` и `long` (не `lon`) в объекте `hotel` внутри каждого результата. Используются для отображения на карте.

## PDF-генератор (`scripts/generate_pdf.py`)

Генерирует путевой документ формата A4 для туристов.

**Входные данные** — JSON-структура:
```json
{
  "meta":       { "order_id", "destination", "dates", "brand" },
  "passengers": [ { "name", "airline", "flight_out", "flight_back" } ],
  "flights":    [ { "label", "date", "dep_time", "arr_time", "dep_city", "arr_city", "info" } ],
  "bookings":   [ { "name", "checkin", "checkout", "address", "booking_id", "food", "deposit", "phone", "note" } ],
  "itinerary":  [ { "day", "date", "subtitle", "color", "stops" | "free_text" } ],
  "notes":      [ { "type": "warn"|"ok", "title", "text" } ]
}
```

**Запуск из CLI:**
```bash
python3 scripts/generate_pdf.py scripts/trip_data.json /tmp/output.pdf
```

**Через API:**
```bash
curl -X POST http://localhost:3000/api/generate-pdf \
  -H "Content-Type: application/json" \
  -d @scripts/trip_data.json \
  --output trip.pdf
```

Тестовые данные — поездка в Калининград: `scripts/trip_data.json`

Следующий шаг: функция `buildTripData(booking)` которая маппит данные из MySQL в этот JSON-формат. Триггер — смена статуса в Битрикс24 или кнопка в админке.

## Флоу заказа (будущее)

```
Пользователь ищет тур
       ↓
Level Travel API → результаты + координаты на карте
       ↓
Клик "Смотреть" → WL Level Travel (russia.mosgortur.ru)
       ↓
Заказ → наша MySQL
       ↓
Заказ → Битрикс24 (CRM)
       ↓
PDF путевого документа → клиенту (email или скачать)
       ↓
Заказ в Личном кабинете
```
