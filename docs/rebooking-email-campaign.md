# Инструкция для email-рассылки: лендинг перебронирования `/rebooking`

Документ для сотрудника, который готовит письма клиентам с отменённой поездкой.

---

## 1. Что это за страница

**Адрес:** https://online.mosgortur.ru/new/rebooking

Это персональная страница для клиента, у которого отменили тур. Клиент:

1. Открывает **уникальную ссылку из письма** (у каждого человека своя).
2. Видит **параметры своей отменённой поездки** (даты, состав, бюджет, номер заявки и сертификата).
3. Внизу — **форма поиска туров Tourvisor** с уже подставленными фильтрами (дата, ночи, люди, дети, бюджет).
4. Выбирает новый тур и **оставляет заявку** в форме Tourvisor (телефон, имя и т.д.).
5. Заявка попадает в нашу систему и далее — в Bitrix24 менеджерам.

**Важно:** отдельной формы «оставить заявку» на странице нет. Клиент оформляет заявку **только через виджет Tourvisor**.

---

## 2. Как устроена ссылка в письме

Базовый формат:

```
https://online.mosgortur.ru/new/rebooking?параметр1=значение&параметр2=значение&...
```

Все данные передаются **в адресной строке** (после знака `?`).  
Каждый параметр — это `имя=значение`, параметры разделяются `&`.

### Пример готовой ссылки (полный набор)

```
https://online.mosgortur.ru/new/rebooking?order=МГТ-2025-04821&cert=СЕРТ-77412&name=Иванов%20Иван%20Иванович&people=5&kids=3&kid1=1&kid2=5&kid3=10&price=185000&nights=10&date=2026-07-14&utm_source=email&utm_campaign=rebooking_july2026
```

> В реальной рассылке кириллица и пробелы в ссылке должны быть **закодированы** (см. раздел 6).

## 3. Параметры: что передавать из базы / Excel

### Обязательный параметр

| Параметр | Обязательно | Откуда брать | Что делает |
|----------|:-----------:|--------------|------------|
| **`order`** | **Да** | Номер заявки в вашей системе, например `МГТ-2025-04821` | Без него страница покажет ошибку «Ссылка некорректна». По этому номеру менеджер потом найдёт клиента в Bitrix. |

### Рекомендуемые параметры (для хорошего UX)

| Параметр | Обязательно | Формат | Пример | Что видит / получает клиент |
|----------|:-----------:|--------|--------|----------------------------|
| **`cert`** | Нет, но желательно | Текст | `СЕРТ-77412` | В карточке: «Сертификат № …». Уходит в заявку менеджеру. |
| **`name`** | Нет, но желательно | ФИО полностью | `Иванов Иван Иванович` | Приветствие: «Здравствуйте, **Иван**!» (берётся **имя** — второе слово в ФИО). |
| `phone` | Нет, но **желательно** | `+79001234567` | Не отображается клиенту. При аннуляции и перебронировании передаётся в Bitrix как телефон из письма; по нему ищется/создаётся контакт. |
| **`people`** | Нет, но желательно | Целое число ≥ 0 | `5` | «5 туристов» в карточке; подставляется в поиск Tourvisor. |
| **`kids`** | Нет | Целое 0–3 | `3` | Сколько детей в поездке. |
| **`kid1`**, **`kid2`**, **`kid3`** | Нет* | Возраст 0–15 лет | `1`, `5`, `10` | Возрасты детей в карточке и в форме Tourvisor (`tv-kid1` и т.д.). |
| **`price`** | Нет, но желательно | Сумма в рублях, целое | `185000` | «Бюджет поездки: 185 000 ₽»; верхняя граница цены в поиске. |
| **`nights`** | Нет, но желательно | Целое > 0 | `10` | «10 ночей»; подставляется в поиск. |
| **`date`** | Нет, но желательно | `ГГГГ-ММ-ДД` или `ДД.ММ.ГГГГ` | `2026-07-14` | Дата начала отдыха в карточке и в поиске. |

\* Если указали `kids=3`, желательно заполнить `kid1`, `kid2`, `kid3` — иначе в Tourvisor возрасты детей могут не подставиться (страница всё равно откроется).

### Параметры для аналитики рассылки (UTM)

Не влияют на отображение страницы, но попадают в заявку для отчётов:

| Параметр | Пример | Назначение |
|----------|--------|------------|
| `utm_source` | `email` | Источник (email, sms, …) |
| `utm_medium` | `newsletter` | Канал |
| `utm_campaign` | `rebooking_july2026` | Название кампании |
| `utm_content` | `variant_a` | Вариант письма / блока |
| `utm_term` | — | Доп. метка (редко) |

## 4. Логика страницы (что происходит после клика)

```
Клиент кликает ссылку в письме
        ↓
Страница читает параметры из URL
        ↓
   order есть? ──нет──→ «Ссылка некорректна» (письмо без номера заявки)
        │
       да
        ↓
Показывается карточка поездки + приветствие по name
        ↓
Подгружается поиск Tourvisor с предзаполненными фильтрами
        ↓
Клиент нажимает «Найти», выбирает тур, отправляет форму с телефоном
        ↓
Заявка сохраняется в нашей админке → затем уходит в Bitrix24
```

### Что предзаполняется в Tourvisor автоматически

| Данные из ссылки | Что делает на странице |
|------------------|------------------------|
| `date` | Даты вылета (от и до — одна и та же дата) |
| `nights` | Количество ночей |
| `people` − `kids` | Число взрослых (только если от 1 до 4) |
| `kids` + `kid1`…`kid3` | Дети и их возрасты |
| `price` | Цена «до … ₽» |

**Поиск туров сам не запускается** — клиент должен нажать кнопку «Найти» в виджете.

### Правила и ограничения

- Если `kids` больше, чем `people` — дети сбрасываются в 0 (страница не ломается, но лучше не допускать в исходных данных).
- Максимум **3 ребёнка** с возрастами (`kid1`–`kid3`).
- Взрослых в Tourvisor можно передать только **от 1 до 4**; при большем `people` число взрослых в виджет может не подставиться (карточка на странице покажет верное значение).
- Направление (страна/курорт) **не** передаётся в ссылке — клиент выбирает его в Tourvisor.

## 5. Минимальная и полная ссылка

### Минимум (только номер заявки)

```
https://online.mosgortur.ru/new/rebooking?order=МГТ-2025-04821
```

Страница откроется, но без приветствия, без карточки с датами/бюджетом и без предзаполнения Tourvisor. **Для рассылки так делать не рекомендуется.**

### Рекомендуемый шаблон для письма

Подставьте данные из строки вашей таблицы:

```
https://online.mosgortur.ru/new/rebooking
  ?order={номер_заявки}
  &cert={номер_сертификата}
  &name={ФИО}
  &people={всего_человек}
  &kids={число_детей}
  &kid1={возраст_1}
  &kid2={возраст_2}
  &kid3={возраст_3}
  &price={бюджет_руб}
  &nights={ночей}
  &date={дата_ГГГГ-ММ-ДД}
  &utm_source=email
  &utm_campaign={название_рассылки}
```

Пустые поля (например, нет детей) — **не добавляйте** в ссылку или ставьте `kids=0` без `kid1`…`kid3`.

## 6. Кодирование ссылки для email-сервиса

Кириллица, пробелы и спецсимволы в URL должны быть закодированы.

| В данных | В ссылке |
|----------|----------|
| Пробел | `%20` |
| `Иванов Иван Иванович` | `Иванов%20Иван%20Иванович` или полное URL-encoding |

**В Excel / Google Sheets:** используйте функцию `ENCODEURL()` для всей строки параметров или готовьте ссылки скриптом (см. ниже).

**В Unisender / Sendsay / Mailchimp:** обычно достаточно вставить ссылку в поле «ссылка» — сервис кодирует сам, если вы вставляете через их редактор гиперссылок.

**Проверка:** откройте ссылку в режиме инкогнито — должны увидеть карточку с вашими данными, а не «Ссылка некорректна».

## 7. Массовая генерация ссылок из CSV

Если рассылка на сотни/тысячи человек, удобнее не собирать ссылки вручную.

### Формат входного файла `input-rebooking.csv`

Колонки (первая строка — заголовок):

```csv
order,cert,name,people,kids,kid1,kid2,kid3,price,nights,date
МГТ-2025-04821,СЕРТ-77412,Иванов Иван Иванович,5,3,1,5,10,185000,10,2026-07-14
МГТ-2025-04822,СЕРТ-77413,Петрова Мария Сергеевна,2,0,,,120000,7,2026-08-01
```

Пустые ячейки для детей — нормально, если `kids=0`.

### Команда (для IT / разработчика)

```bash
node scripts/generate-rebooking-links.js input-rebooking.csv output-rebooking-links.csv
```

На выходе — тот же CSV + колонка **`link`** с готовой персональной ссылкой на каждую строку. Эту колонку подставляют в поле «ссылка на кнопку» в сервисе рассылки.

## 8. Текст кнопки в письме (рекомендации)

Примеры:

- «Подобрать новый тур»
- «Выбрать тур вместо отменённой поездки»
- «Перейти к подбору тура»

В письме можно кратко пояснить:

> Мы уже учли параметры вашей поездки — останется выбрать подходящий вариант и оставить заявку.

## 9. Частые ошибки

| Ошибка | Последствие |
|--------|-------------|
| Нет `order` в ссылке | «Ссылка некорректна» |
| Опечатка в `order` | Заявка не привяжется к нужному клиенту в CRM |
| Ссылка ведёт на `online.mosgortur.ru/new/rebooking` без `?` | Ошибка |
| ФИО не закодировано / обрезано почтовиком | Нет приветствия или битая ссылка |
| `date` в формате `14/07/2026` | Дата может не распознаться — используйте `2026-07-14` или `14.07.2026` |
| Одна общая ссылка на всех | Нет персонализации, в CRM не будет номера заявки |

## 10. Чеклист перед отправкой рассылки

- [ ] В каждой строке рассылки уникальный `order`
- [ ] Есть `cert` и `name` (желательно)
- [ ] Заполнены `people`, `price`, `nights`, `date` — если они известны
- [ ] Для детей: `kids` + возрасты `kid1`…`kid3`
- [ ] Добавлены UTM (`utm_source=email`, `utm_campaign=…`)
- [ ] 2–3 тестовые ссылки открыты в браузере — карточка и Tourvisor выглядят правильно
- [ ] Тестовая заявка в Tourvisor дошла до менеджеров (по согласованию с IT)

## 11. Тестовая ссылка для проверки вёрстки письма

```
https://online.mosgortur.ru/new/rebooking?order=МГТ-TEST-EMAIL&cert=СЕРТ-TEST-01&name=Тестов%20Иван&people=5&kids=3&kid1=1&kid2=5&kid3=10&price=185000&nights=10&date=2026-07-14&utm_source=email&utm_campaign=test
```

Ожидаемое поведение:

- Приветствие: «Здравствуйте, Иван!»
- В карточке: 14 июля 2026, 10 ночей, 5 туристов, бюджет 185 000 ₽
- Внизу: «Заявка № МГТ-TEST-EMAIL · Сертификат № СЕРТ-TEST-01»
- В Tourvisor предзаполнены дата, ночи, состав и цена
