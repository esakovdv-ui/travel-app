# Рассылка Bitrix из Excel-выгрузки (перебронь / аннуляция)

Инструкция для кампании с одной персональной ссылкой на `/rebooking`.  
На лендинге клиент сам выбирает: **подобрать новый тур** или **аннулировать заявку**.

---

## Шаг 1. Сгенерировать CSV из Excel

```bash
cd "/Users/dima/Downloads/Travel app"
node scripts/generate-rebooking-campaign-xlsx.js "/path/to/выгрузка.xlsx"
```

Или с явным путём выхода:

```bash
node scripts/generate-rebooking-campaign-xlsx.js \
  "/Users/dima/Downloads/крым_срочные_июль_полный.xlsx" \
  "/Users/dima/Downloads/крым_bitrix.csv"
```

На выходе CSV с колонками:

| Колонка | Для чего |
|---------|----------|
| `email` | Адрес получателя в рассылке |
| `name` | Имя в письме |
| `phone` | Справочно + **в ссылке** | Телефон из Б24; попадает в query `phone=…` персональной ссылки |
| `order` | Номер заявки |
| `deal_id` | ID сделки B2C из колонки «Б24 ссылка» |
| `rebooking_link` | **Персональная ссылка на лендинг** — главное поле для кнопки в письме |
| `utm_campaign` | Метка кампании (`krym_srochnye_july2026` по умолчанию) |

Переменные окружения (опционально):

```bash
UTM_CAMPAIGN=krym_srochnye_july2026 \
UTM_SOURCE=email \
UTM_MEDIUM=newsletter \
node scripts/generate-rebooking-campaign-xlsx.js file.xlsx
```

**Готовый файл для июльской выгрузки Крым:**

`/Users/dima/Downloads/крым_срочные_июль_полный_bitrix_рассылка.csv` — **320 получателей**.

## Шаг 2. Поле в Bitrix для ссылки (один раз)

**CRM → Настройки → Настройки CRM → Поля → Сделки**

Создайте строковое поле, например:

- Название: `Ссылка перебронирования`
- Код: `UF_CRM_REBOOKING_LINK`

(Если поле уже есть — используйте его.)

---

## Шаг 3. Импорт ссылок в сделки

**CRM → Сделки → Импорт**

1. Загрузить `крым_срочные_июль_полный_bitrix_рассылка.csv`
2. Сопоставить колонки:

| Колонка CSV | Поле Bitrix |
|-------------|-------------|
| `deal_id` | **ID сделки** (ключ обновления) |
| `rebooking_link` | `UF_CRM_REBOOKING_LINK` |
| `email` | Email контакта (если импорт обновляет контакт) |

3. Режим: **обновить существующие** сделки по ID.
4. Запустить импорт.

> Если импорт по `deal_id` недоступен — попросите IT обновить поле через REST или импорт контактов по `email` + ручная привязка. В вашей выгрузке `deal_id` есть у всех 320 строк.

## Шаг 4. Шаблон письма в Bitrix

**Маркетинг → Email-рассылки → Создать**

Текст (пример):

```
Здравствуйте!

Ваша поездка в Крым отменена. На вашей персональной странице вы можете:
• подобрать новый тур с учётом параметров поездки;
• запросить аннуляцию заявки.

[  Открыть мою страницу  ]
```

Кнопка — **одна**, ссылка из поля сделки:

- В редакторе: **Персонализация** → поле сделки `UF_CRM_REBOOKING_LINK`
- Или макрос вида `{=Document:UF_CRM_REBOOKING_LINK}` (зависит от версии Bitrix)

**Не вставляйте одну общую ссылку** — у каждого клиента свой `order` в URL.

---

## Шаг 5. Сегмент получателей

Вариант A — **по сделкам** (рекомендуется для этой выгрузки):

1. Сегмент: сделки из воронки B2C, ID из выгрузки / стадия «отмена» / список из импорта.
2. В письме подтягивается `UF_CRM_REBOOKING_LINK` из карточки сделки.

Вариант B — **импорт получателей в Маркетинг**:

1. Маркетинг → Адреса → Импорт CSV (`email`, `name`, доп. поле `rebooking_link`).
2. В шаблоне — макрос доп. поля со ссылкой.

---

## Шаг 6. Проверка перед массовой отправкой

1. Возьмите **2 строки** из CSV (себя + коллегу).
2. Откройте `rebooking_link` в браузере — страница с параметрами поездки.
3. Проверьте кнопку **«Аннулировать»** → карточка в смарт-процессе 1302, столбец **«Аннуляция»**; в комментарии — **Телефон (из письма)** из колонки Excel.
4. Проверьте **Tourvisor** → заявка в админке [rebooking-admin](https://motrip.ru/rebooking-admin) → смарт-процесс **«Перебронь»**.

## Что попадает в ссылку из вашей выгрузки

| Из Excel | В URL |
|----------|--------|
| Номер заявки | `order` |
| ФИО (Б24) | `name` |
| Телефон (Б24) | `phone` (формат `+7…`) |
| Дата выезда | `date` (2026-07-01) |
| Туристов | `people` |
| UTM | `utm_source`, `utm_medium`, `utm_campaign` |

Цену поездки в ссылки **не включаем**.

В выгрузке **нет** сертификата, детей и ночей — страница откроется и без них. Если появятся колонки `Сертификат`, `Детей`, `Ночей` — скрипт подхватит автоматически.
